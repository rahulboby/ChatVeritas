Abstract— Dirty and inconsistent data remain one of the main challenges in data analysis and machine learning systems, where issues such as incomplete data, duplicate records, anomalies, and inconsistencies tend to reduce analytical reliability and model performance. Existing tools address specific problems separately, but lack an integrated framework for complete data quality assessment. We present DataVeritas, a system for unified data quality assessment across multiple dimensions of dataset quality such as auditing, anomaly detection, rule-based consistency validation and entity resolution into one combined workflow. To improve assessment clarity, DataVeritas introduces a trust score system that summarizes various quality metrics into one interpretable measure. To demonstrate the framework’s ability to identify data quality issues across multiple dimensions, experimental evaluation using real-world tabular datasets is implemented, proving the framework’s ability to identify and treat various aspects of data quality issues, such as outliers, near-duplicate records, incomplete records, and logical inconsistencies. The approach promotes the use of human-in-the-loop approaches to entity resolution as well as domain-based consistency checks that help perform controlled data cleaning. Compared to current state-of-the-art stand-alone applications such as ydata-profiling, Great Expectations, and OpenRefine, DataVeritas increases the coverage of data quality capability integration by up to 133%-250%.

I. INTRODUCTION
Data analysis-based and machine learning models are very sensitive to the quality of input data [1], [2]. However, there are various problems associated with datasets including missing data, duplications, contradictions, and outliers [3]. Such problems may adversely affect subsequent operations; duplications may affect the outcomes of aggregations, whereas outliers can adversely affect statistics and the learning process itself [4]. The outcome is that the predictions provided by data models and information extracted from them is incorrect, leading to flawed decision making [5]. Therefore, maintaining good quality data before analysis is critical [6].
Many approaches have been created to handle different aspects of data quality and data exploration. Tools such as Pandas Profiling (ydata-profiling) automatically analyze data distributions, identify missing data, and compute correlations but cannot perform any duplicate resolution or entity resolution, nor do they offer any scoring functionality. Great Expectations offers a tool for validation of data based on user-defined rules but again requires defining such rules manually, does not have scoring or entity resolution, and does not provide any automatic data exploration capabilities. OpenRefine helps to clean up data and perform clustering tasks but requires lots of manual work from the user and lacks both automation and the capability of doing end-to-end audits. In other words, the current state of affairs implies that the available tools function separately and tackle only certain issues related to data quality.
This paper proposes the framework DataVeritas, which offers an end-to-end approach for the evaluation of the quality of tabular data sets. It incorporates various aspects of data quality assessment in a modular pipeline fashion that allows for the simultaneous evaluation of data quality. Multi-dimensional auditing, detection of duplicates and anomalies, entity resolution, computation of the trust score, and consistency checks using rules are some of the functionalities offered by this solution, which aims to address the aforementioned challenges.
Through DataVeritas, multiple dimensions of data quality, including completeness, cardinality, uniqueness, consistency, and multivariate outlier detection, are embedded in a single trust-scoring system, allowing for a clear understanding of the level of reliability of a particular dataset. While it enables independent analysis and evaluation of each component of the data quality equation, it offers a consolidated trust score as well to ease up the process of assessing and prioritizing datasets. Furthermore, it features an entity resolution module, combining the capabilities of detecting exact duplicates and applying edit distance algorithms for fuzzy matching methods for spotting near-duplicate entries caused by typos or inconsistent typing. Besides, DataVeritas features a human-in-the-loop approach, where users can adjust similarity criteria when resolving entries.

II. RELATED WORK
Various profiling tools [7]-[11] and automated validation techniques have been suggested and used in practice to solve certain data quality issues. For instance, various Python-based profiling tools, including Pandas ydata-profiling authored and maintained by YData Labs Inc., performs exploratory data analysis using static HTML profiling reports, which include distributions, missing value analysis, correlation, and statistical summaries. Similarly, Sweetviz creates visualization reports explaining relationships, correlations, and dataset comparisons. These tools help users gain insights into the structure and characteristics of datasets effectively, but focus on descriptive analysis only and do not integrate duplicate resolution, entity resolution, trust score computation, or comprehensive quality management functionality. Besides, almost all these profiling frameworks use Python and work mostly with Pandas DataFrames.
MAN Alpha Technology introduced D-Tale, a Python-based library that provides a web-based interactive interface allowing sorting, filtering, and editing of tabular data. It makes exploratory data analysis more convenient, but does not integrate features related to automated auditing, quality assessment, and improvement of data.
Francois Bertrand’s Great Expectations and Deequ developed by a team in Amazon Web Services allow users to validate the schemas and other constraints on datasets. These tools allow defining rules for missing values detection, data type, range, and consistency checking. Despite being efficient for enforcing certain predefined data constraints, they require significant manual specification of domain-specific rules and knowledge of the business domain.
The use of data cleaning and entity-resolution tools like Aidi Ahmi’s OpenRefine includes features such as clustering, transforming, normalizing, and correcting inconsistent records. However, they mainly involve extensive human effort and lack integrated support for auditing, scoring, anomaly detection, and trust evaluation.
While several data quality management systems have been developed and can be used for certain aspects of DQM, most of them act individually and focus on performing certain tasks alone, such as profiling, validation, visualization, and data cleaning [12]. However, there is a lack of integration of systems that can help in exploring audits, scoring trust, detecting anomalies, removing duplicates, and validating rules. The DataVeritas framework is intended to bridge this gap [13]. Table I shows a comparison of different systems with DataVeritas on various data quality validation and remediation capabilities.
Several new frameworks that rely on automation and machine learning have emerged as solutions to certain aspects of data quality improvements. HoloClean [14] is a framework based on probabilistic inference, that uses integrity constraints to automate the process of error identification and fixing. BoostClean [15] focuses on the aspect of improving machine learning performance by automating the data cleaning process. ActiveClean [16] proposes iterative data cleaning techniques and models where cleaning and training can be conducted simultaneously, and Raha [17] proposes techniques for rule-supported error detection for structured data.
While such techniques achieve high levels of automation in certain areas of improvement, they are not designed to provide an integrated framework covering multiple steps such as exploratory analysis, trust score calculation, anomaly detection, entity resolution, and rule-based error detection. DataVeritas, however, emphasizes multi-dimensional data quality assessment and governance-oriented interpretability. 
III. METHODOLOGY
A. System Overview
The DataVeritas framework follows a unified architecture designed to perform end-to-end data quality assessment and remediation on tabular datasets. The workflow begins with data ingestion as a file, followed by auditing, anomaly and duplicate detection, entity resolution, consistency validation through user-given rules, and trust score computation. The outputs are then visualized through an integrated reporting and monitoring interface to assist the user in understanding reliability and quality issues.
The architecture is divided into mainly three layers: (1) the user interaction layer, (2) the feature processing layer, and (3) the core computation layer. The user interaction layer provides the ability for dataset upload, visualization, validation, and quality assessment. The feature layer contains multiple modules for visualizing and profiling the data in different aspects of data quality, like duplicate detection, anomaly detection, cardinality, and completeness. The core layer performs computational operations by implementing statistical analysis, similarity computation, scoring aggregation, and validation processing. Figure 1 represents the System Architecture of DataVeritas.
The design of the framework enables the independent execution of different auditing components while maintaining an integrated workflow for comprehensive dataset evaluation.  
B. Data Quality Auditing Framework
The framework performs multi-dimensional auditing of tabular datasets using various statistical analysis and rule-based evaluation techniques [18], [19]. 

Multiple quality dimensions are evaluated to give a score for each of those quality checks, all of which contribute to the final trust score.
Completeness audit is aimed at evaluating the percentage of missing values in attributes and determining columns that have incomplete rows. Such an analysis will help understand the presence of possible problems that will arise due to missing data during further analyses.
Non-Null Score for each column is computed by:
	NN_j=\frac{N_{non\mathrm{-}null}^{\left(j\right)}}{N_{total}^{\left(j\right)}}	(1)

Completeness score for the entire dataset is computed by:
	C=\frac{N_{non\mathrm{-}null}}{n\times m}	(2)
Where n: number of rows, m: number of columns, N_{non\mathrm{-}null} is the total populated cells.
Uniqueness audit is performed to detect duplicate records and determine attribute-level redundancies [20], [21]. Duplicate data will be analysed in order to determine the extent to which it causes conflicts related to redundant entries.
Consistency auditing will be performed to analyse possible logical conflicts and invalid connections between attributes. Thus, possible contradictions in categorical values and constraints in numerical values will be detected.
	S=1-\frac{N_{violations}}{N_{evaluated}}	(3)
Cardinality analysis evaluates the distribution and variety of categorical attributes by calculating unique values in columns. Such an analysis will help detect abnormal distribution of values and excessive categorization of data.
	CR=\frac{N_{unique}}{N_{records}}	(4)
Statistical distribution analysis will be done to analyze skewness and imbalance of numerical attributes and other inconsistencies associated with them.
C. Data Trust Score Model
To simplify dataset evaluation, the system introduces a composite Trust Score that aggregates multiple quality metrics into a single reliability score. Since different quality dimensions may contribute unequally to the overall dataset quality, the framework computes weighted scores across multiple auditing categories, with the ability to assign higher weights to more impactful issues user-configurable weights. 
As default, the weights are assigned based on heuristic assessment of relative impact of each dimension on data reliability, though in future implementations may incorporate adaptive or advance weighting strategies. 
The Data Trust Score is calculated as:
	T=\frac{\sum_{i=1}^{\operatorname{n}}{\operatorname{w}_i\operatorname{q}_i}}{\sum_{i=1}^{\operatorname{n}}\operatorname{w}_i}	(5)
Where: T denotes the final Trust Score, wi represents assigned weights for each quality dimension i, qi represents the normalized score of that dimension.
It is ensured that the final trust score remains between 0 and 1 regardless of the absolute magnitude of assigned weights.
D. Duplicate Detection and Entity Resolution
The system consists of a hybrid pipeline for detecting duplicates and entity resolution [22], which is meant to detect duplicates as well as near-duplicates that may arise because of inconsistencies in formatting and spellings. The first step in the pipeline involves exact matching between records for the selected fields to find the potential duplicates. Detecting exact duplicates requires minimal processing time and helps in finding complete duplicates in the datasets.

To identify near-duplicate records, the framework additionally performs edit-distance-based similarity analysis using Levenshtein distance and fuzzy matching techniques [23]. These methods enable the identification of records containing spelling variations, typographical inconsistencies, and partially matching values. Phonetic matching techniques are also being explored as future enhancement to identify records with similar pronunciations but different spellings [24]. This improves duplicate identification in datasets containing human-generated text inconsistencies. Figure 2 represents the full workflow of the Entity Resolution Engine.
Levenshtein Distance Recurrence:
lev\left(i,j\right)=\left\{\begin{matrix}\max{\left(i,j\right)},&\mathrm{if\ }\min{\left(i,j\right)}=0\\\min{\left(\begin{align}lev\left(i-1,j\right)+1,\;\\lev\left(i,j-1\right)+1,\;\\lev\left(i-1,j-1\right)+cost\end{align}\right)},&\mathrm{otherwise}\end{matrix}\right.	(6)
Normalized Levenshtein similarity:
	Sim_{lev}=1-\frac{lev\left(s_1,s_2\right)}{\max{\left(\left|s_1\right|,\left|s_2\right|\right)}}	(7)
The framework further implements a weighted mechanism that enables users to configure field-specific similarity weights and thresholds for similarity checks, which would be used when computing the similarity score.
To reduce risks associated with blind automated remediation and merging, the framework incorporates a human-in-the-loop system where users can review candidate matches prior to record merging and correction [25].
E. Anomaly Detection Framework
DataVeritas performs anomaly detection using both statistical and model-based techniques to identify abnormal observations at both attribute-level and record-level.
Outliers are computed using the condition given below.
	LB=Q_1-1.5\times IQR	(8)
	UB=Q_3+1.5\times IQR	(9)
	x < LB  ∨  x > UB	(10)
Anomaly Score is computed using
	A=1-\frac{N_{anomaly}}{N_{records}}	(11)
Statistical outlier analysis is mainly performed for univariate outliers by using Interquartile Range (IQR)-based detection methods [26]. This approach is done for every numerical attribute. For detecting anomalies across multiple features, the framework integrates the Isolation Forest algorithm for simple multivariate anomaly detection. Isolation Forest [27], [28] is lightweight, highly efficient and scalable algorithm that can be implemented for high-dimensional data through an unsupervised algorithm with minimal memory requirements.

F. Rule-Based Consistency Validation
The framework incorporates a configurable rule-based consistency validation engine for identifying logical inconsistencies and invalid data based on user-defined rules and relationships within datasets. Users are allowed to define custom validation rules based on domain-specific constraints and dataset requirements. The validation engine supports rules for data type constraints, numerical ranges, categorical consistency checks, null restrictions, and relational conditions between fields. In data auditing, these defined logical conditions and rules are implemented to compute the consistency score.
The configurable design of the validation engine enables adaptation across multiple domains and allows users to customize quality constraints according to specific business or analytical requirements.
G. System Implementation
DataVeritas is implemented using Python [29] and Streamlit [30] for interactive Data Quality assessment and visualization. Core data processing operations are performed using Pandas [31] and NumPy [32], while anomaly detection operations use machine learning models from the Scikit-learn library [33]. Independent auditing components communicate through a shared workflow and session state management. Caching and state preservation improve responsiveness and reduce redundant computations during refreshing, or repeated computations. 
The implementation enables extensibility for integrating additional modules, tools, and validation strategies in future versions of the framework.
IV. EXPERIMENTAL RESULTS
DataVeritas was tested using a real-world sales pipeline dataset [34] containing approximately 8,800 records and 8 attribute consisting of both numerical, categorical and date fields. These features included information on sales opportunities, customer accounts, sales representatives, product lines, financial amounts, and time periods of interaction. In essence, this dataset already had inconsistencies like missing data, outliers, and violated rules, making it a good candidate for the evaluation of the suggested framework.

Different modules and tools of the framework were evaluated with the same dataset to maintain consistency throughout the experimentation process. For evaluating the entity resolution system, a smaller modified subset containing intentionally injected inconsistencies like missing values, near-duplicate errors and outliers was additionally used in order to demonstrate fuzzy matching and similarity-based duplicate detection capabilities.

A. Data Quality Auditing Results
The auditing framework successfully identified multiple inconsistencies and reliability issues in the dataset. The completeness audit identified a significant number of incomplete records, particularly within the close_date and close_value fields. Column health and record integrity based on missing value distributions and completeness scores were also classified. Figure 5 demonstrates the result of the completeness audit section showing the amount of missing data in each field of the dataset.

Cardinality analysis identified certain variables that exhibited a significant variation in their value distribution. Specifically, the variable opportunity_id exhibited very high cardinality due to the existence of unique transaction codes, while other categorical variables such as deal_stage and product had a less diverse distribution. Such analysis helped identify any abnormal distribution pattern of features.
Outlier detection in the system was used to detect unusual numeric data from the close_value feature. Using statistical tests, it was observed that there were several values that were not operating within the expected range. In addition, the model used box plot visualization for distribution analysis to identify outliers. Figure 6 shows a box plot that represents the distribution of values in the sample column close_value, using whiskers and boxes, and points for outliers.
The generated audit results demonstrate the ability of the framework to identify multiple dimensions of data quality degradation using integrated profiling and statistical analysis mechanisms.
B. Trust Score Evaluation
The Data Trust Score model integrated the four quality measures into one reliability score that reflected the structural integrity of the dataset. The analysed dataset obtained a high reliability score of 91.5% despite missing data and data validation issues.
Component-based analysis revealed differences in the reliability of different quality factors. Both the uniqueness and consistency indicators recorded high scores because of the lack of duplicate data and correct structural dependencies; however, outlier detection and non-null record evaluation obtained lower scores because of missing data and irregular numeric values. C. Entity Resolution Evaluation
The entity resolution architecture was assessed using the reduced subset, which had deliberately created inconsistency issues in the textual data and near duplicates. The entity resolution architecture made use of exact matching, fuzzy similarity techniques, and weighted similarity calculation to find out duplicate entities caused by spelling differences and inconsistencies.
The weighted similarity approach allowed flexible attribute weight assignment during the process of duplicate detection. Adjustments could be made to similarity thresholds and attribute weights using the configurable interface provided by the system.
To assess the contribution of different entity resolution methods, component-level experiments using exact matching and fuzzy matching thresholds were conducted independently. Exact matching identified 68 candidate pairs with no ambiguity, while fuzzy matching at 90% gave the 68 pairs for auto-merging from exact matches, plus 4 pairs for manual evaluation. The system was able to detect near duplicates that had differences in terms of capitalization, spacing errors, and slight misspellings. The manual intervention in validating potential duplicate data further ensured that errors resulting from automated merges were minimized through user verification. V. LIMITATIONS
This suggested approach has certain shortcomings that have to be addressed, though the efficiency of the system cannot be overlooked. The system functions most efficiently with the use of small to medium data; the performance degrades when working with large datasets. In terms of rule-based validation, the use of pre-defined rules is required for evaluation of data quality, but their development is manual and requires user input; hence, it cannot be considered automatic. The framework currently operates on static datasets, meaning that it is not applicable to live streams. Also, it is designed for human use on screen and thus is not currently optimized for automated production pipelines. The model works best with UIs and is not incorporated in the automation process of the production pipeline entirely.
Large-scale industrial validation involving enterprise-scale tabular datasets was not included in the present evaluation, as the primary objective of this work was to demonstrate the feasibility of a unified data quality assessment framework through architectural integration, modular auditing, and trust score computation, rather than conduct extensive production-scale benchmarking.
Accordingly, the experimental evaluation focused on validating the functional correctness and interoperability of the proposed modules using medium-scale datasets containing representative real-world data quality issues such as missing values, inconsistencies, anomalies, and duplicate records. Broader empirical validation using enterprise-scale industrial datasets and distributed computing environments remains an important direction for future work.
While the current version of DataVeritas is intended as an interactive prototype rather than a production-ready system, the modularity of its design facilitates its potential adoption in more complex, distributed data governance environments. In addition, lightweight auditing functions such as completeness checks, consistency checks, and uniqueness analysis can be incorporated within ETL or streaming data governance workflows. Computationally intensive processes such as entity resolution would require scalable parallel or distributed computation to efficiently handle large industrial datasets.

VI. CONCLUSION
DataVeritas presents a unified framework for complete data quality assessment tabular datasets by integrating multiple auditing and validation tools within a single system. The proposed system combines assessment of multiple dimensions of data quality such as anomaly detection, rule-based validation, consistency checks, entity resolution, etc. into one interpretable trust score that acts as an interpretable aggregation of all the dimensions. By combining functionalities that are usually fragmented across different tools, the framework simplifies data preprocessing and improves analysis of data quality prior to downstream analytical and machine learning tasks, through an interactive and modular architecture. Experimental evaluation demonstrates the framework’s ability to identify missing values, anomalies, inconsistencies, exact duplicates and near-duplicate records across real-world datasets, showcasing its effectiveness as a practical and efficient preprocessing and auditing solution. The current implementation is best suited for small to medium datasets. Hence, future work should focus on improving scalability through optimized processing pipelines and more efficient duplicate comparison approaches for larger datasets.  Further improvements could also include AI-based suggestions for validation rules, applying machine learning techniques in entity resolution for better near-duplicate detection, and smart remediation suggestions for detected data quality issues.
